
# 项目2：基于Transformer的生成式序列推荐

> 核心价值：将推荐任务重构为序列生成任务（用户历史→下一物品），验证生成式召回新范式，为OneRec架构重构提供实践基础。

---

## 🎯 技术方案

### 1. 问题定义与范式创新

传统召回的问题：ItemCF、双塔等依赖ID共现，新物品没有交互记录就无法被推荐——这就是“冷启动”问题的根源。

生成式召回：不依赖ID共现，而是学习“用户历史物品序列 → 下一物品”的生成模式，天然支持冷启动。

### 2. 核心组件

| 组件 | 作用 |
|------|------|
| Token Embedding | 将物品ID映射为连续向量 |
| 位置编码 | 保留序列中的顺序信息 |
| Transformer Decoder | 捕捉序列中的长程依赖关系 |
| 因果掩码（Causal Masking） | 保证生成时只依赖历史信息，符合推荐系统的时序约束 |
| 输出层 | 将最后一维映射回物品ID空间 |

### 3. 样本构造策略

- 滑动窗口：对每个用户的历史序列，用长度为10的窗口滑动截取，生成（输入序列 → 下一物品）的训练对
- 数据规模：从MovieLens 100K生成5.4万条训练样本

### 4. 评估指标

- Recall@K：推荐列表Top-K中命中用户真实下一物品的比例
- 对比基线：随机猜测 ≈ 1/物品总数 ≈ 0.07%

---

## 📊 核心指标

| 指标 | 结果 |
|------|------|
| Recall@5 | 27.36% |
| Recall@10 | 38.41% |
| 随机基线 | < 0.1% |

Recall@10 = 38.41%意味着每10次推荐中有近4次命中用户真实感兴趣的下一个物品——验证了生成式范式在序列推荐中的可行性。

---

## 🔧 技术栈

- 框架：PyTorch
- 核心模块：`nn.TransformerDecoder`、`nn.MultiheadAttention`
- 数据处理：NumPy、Pandas
