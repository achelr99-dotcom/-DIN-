# 项目3：大模型推荐轻量化部署与RLHF方案设计

> 核心价值：设计“LoRA微调 + 4-bit量化 + GGUF压缩 + FastAPI服务”完整工程链路，并探索DPO实现偏好对齐——覆盖大模型从适配到部署再到对齐的全流程。

---

## 🎯 技术方案

### 1. 高效训练与推理

| 技术 | 作用 | 效果 |
|------|------|------|
| LoRA微调 | 参数高效微调，只训练适配器层 | 可训练参数 < 1% |
| 4-bit量化 | 降低模型存储和计算精度 | 8倍模型压缩（FP32→4-bit） |
| GGUF压缩 | 进一步优化推理速度 | CPU可运行 |
| FastAPI | 封装为RESTful服务 | 支持实时调用 |

全链路：基座模型（Qwen2.5-0.5B）→ LoRA微调 → 4-bit量化训练 → GGUF压缩 → FastAPI部署

### 2. LLM知识对齐

通过LoRA微调，将推荐任务转化为文本生成任务，让大模型学习“用户历史评论 → 推荐理由”的映射，实现大模型知识与推荐目标的迁移。

### 3. RLHF偏好对齐

在监督微调基础上，引入强化学习：

| 要素 | 设计 |
|------|------|
| 奖励信号 | 用户点击、完播率、停留时长（隐式反馈） |
| 算法 | DPO（Direct Preference Optimization）——比PPO更稳定、无需训练奖励模型 |
| 目标 | 模型输出与用户真实偏好对齐，避免监督微调脱离业务目标 |

### 4. 工业级扩展性

| 维度 | 扩展方案 |
|------|---------|
| 基座模型 | Qwen2.5-0.5B → LLaMA-3B/7B |
| 数据规模 | 千级 → 百万级 |
| 架构 | 无需重新设计，直接替换即可 |
