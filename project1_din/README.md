# 项目1：基于DIN的序列CTR预估

> 核心价值：从零实现DIN（Deep Interest Network），用目标注意力机制解决用户历史兴趣的稀疏激活问题，验证排序阶段注意力机制的有效性。

---

## 🎯 技术方案

### 1. 问题定义
在推荐系统的 【排序阶段】，需要预估用户对当前候选物品的点击率（CTR）。用户历史行为序列中，不同物品对当前预测的贡献不同——用户买过手机，不一定对当前展示的耳机感兴趣。

### 2. 核心创新：目标注意力池化（Target Attention Pooling）

传统做法是对用户历史物品做平均池化，所有历史行为权重相同，导致兴趣被“平均化”淹没。

DIN的做法：引入目标注意力机制，对每个候选物品动态计算用户历史中每个物品的注意力权重：

- 输入：用户历史物品序列 + 当前目标物品
- 核心计算：对历史序列中的每个物品，与目标物品拼接后过MLP得到注意力分数
- 输出：加权聚合后的用户兴趣表征

### 3. 训练策略

- 负采样：设计1:1负采样策略（每个正样本配一个随机负样本），解决正负样本不平衡问题
- 数据规模：MovieLens 100K，构建10.8万条正负样本对
- 评估指标：AUC（业界CTR预估标准指标）

---

## 📊 核心指标

| 指标 | 结果 |
|------|------|
| AUC | 0.8983 |
| 随机基线 | 0.5 |
| 样本量 | 10.8万（正负1:1） |

AUC 0.8983意味着模型对正负样本的区分能力极强，远高于随机猜测，验证了注意力机制在序列建模中的有效性。

- 数据处理：NumPy、Pandas
- 评估：Scikit-learn（AUC计算）
